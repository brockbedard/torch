# Ember Eight — Playbook Spec

**Status:** Locked design 2026-04-14. Drives the rewrite of 8 `*Plays.js` files.
**Companion to:** `TORCH-EMBER-EIGHT-DESIGN-PROPOSAL.md` (overall plan + 7-on-7 meta insights).

## Per-team play counts

Each team gets ~20 plays total = ~12 offensive concepts + ~8 defensive packages, matching the existing per-team file size.

## 7-on-7 format guardrails (locked from research)

- **Run game (3 OL only):** GH Counter is the king. Inside Zone weakened — must be Split Zone (TE slice). Buck Sweep needs a TE puller. Counter GT and Power-with-FB don't translate.
- **Pass game (4-sec throw clock + 1 LB box):** Mesh / Snag / Stick / Shallow Cross dominate. Deep Dig and Deep Comeback are obsolete. Glance and Stick RPOs are lethal. Perimeter screen RPOs die (no tackle hinge-block).
- **Defense:** Simulated Pressures (Creepers) are THE weapon. Tampa 2 only works with a DL drop. Cover 2 Man and traditional zone blitzes from a 2nd LB are impossible.

## Play data shape (for code generation)

Each play in `*Plays.js` will follow the existing schema:

```js
{
  id: 'lar_o1',           // team prefix + sequential
  name: 'GH Counter',
  type: 'RUN',            // RUN | PASS | RPO | SCREEN | TRICK | SPECIAL
  formation: '11 Pistol',
  base: 5,                // base yard expectation
  variance: 3,            // standard deviation
  success: 0.55,          // success probability
  // Optional: vsRun/vsPass modifiers, success-down conditions, etc.
}
```

Base yard / variance / success values get tuned during balance testing — the doc just locks the play *identities* per scheme. Tuning happens via `balanceTest.js` runs after the initial code lands.
